01 Network
🌐
Physical NIC
enp1s0f0 · wlp2s0 · promiscuous mode
Raw packet stream from the physical network. Both wired (enp1s0f0) and wireless (wlp2s0) interfaces supported. Configured via NETWORK_INTERFACE in .env. Wireless does not support true promiscuous mode in managed mode — use wired for full visibility.

raw packets
02 Capture
🔴
Suricata 7.0.8
Signature IDS · EVE JSON · auto rule updates
Signature-based IDS. Inspects packets via af-packet and generates EVE JSON alerts and rich metadata. Background watchdog runs suricata-update every 24 h with zero-downtime USR2 rule reload. EVE rotated on size via HUP signal.
eve.json community-id suricata-update network_mode: host USR2 reload NET_ADMIN + NET_RAW
🟠
Zeek 7.0.4
Protocol analyzer · conn/dns/ssl/http · hourly rotation
Deep packet inspection producing structured JSON logs per protocol: conn, dns, ssl, http, dhcp, files… Logs rotate hourly. MAC addresses via mac-logging policy. Cross-tool correlation with Suricata via shared community_id flow hash.
conn.log dns.log ssl.log community-id hourly rotation MAC logging network_mode: host
JSON logs → /var/log/ids/
03 Pipeline
🔷
Vector.dev 0.53
VRL transforms · hourly NDJSON partitions · stateless
Reads Suricata EVE + Zeek JSON, normalises via VRL transforms, stages as hourly-partitioned NDJSON files. Stateless by design — all schema control and TTL management stays in the Python manager. Past-hour files are immutable, current-hour grows.
VRL transforms hourly partitions NDJSON staging stateless alpine image
bulk-load NDJSON · read_json_auto()
04 Storage
🗄️
DuckDB Manager
24 hr TTL · OUI/GeoIP enrichment · 7 anomaly detectors · nmap
Central data engine. Ingests NDJSON → ids.duckdb. 24 hr TTL purge + VACUUM. IEEE OUI manufacturer lookup (~32 K entries) + DB-IP GeoIP country lookup (~250 K IPv4 ranges). Materializes devices and external_ips every 5 min. Detects 7 anomaly types with EMA behavioral baselines. Scheduled weekly nmap scan. Copies read-only snapshots for each consumer. Threat Intel RAG indexing via rag.duckdb.
events devices external_ips anomaly_events nmap_scans OUI lookup GeoIP lookup 24 hr TTL EMA baselines rag.duckdb 7 anomaly detectors
read-only DuckDB snapshots → each consumer

Streamlit Chat
NL queries · 12 tools · nmap · :8501
"Chat with your Network" — natural language queries via Ollama tool-calling loop. 12 tools: query_events, get_devices, get_alerts, get_external_connections, get_dns_top_domains, get_traffic_by_protocol, check_whitelist, nmap_scan, send_notification, send_email, get_scan_history, rag_search. RFC1918-only nmap enforcement. SELECT-only DuckDB safety.

🚨
Alert Agent
Anomaly poll · LLM email · 60 s
Polls anomaly_events every 60 s. Feeds each anomaly to Ollama, which optionally queries additional context, then drafts and sends a rich email alert. State tracked in alert_state.db via composite key (anomaly_id, detected_at) — survives DuckDB sequence resets. RAG context pre-injected for Suricata alerts.
60 s poll Gmail SMTP LLM-drafted alert_state.db RAG pre-inject 7 anomaly types
Ollama API · tool calls · results
06 AI
🧠
Ollama + Qwen2.5
qwen2.5:3b · nomic-embed-text · :11434 · host process
Host-native LLM inference (no container). qwen2.5:3b for chat — ~20 s/query, 2 GB RAM, fits RPI5. nomic-embed-text for Threat Intel RAG embeddings. OpenAI-compatible tool/function calling. Streamlit and Alert Agent share this single endpoint at localhost:11434.
:11434 qwen2.5:3b nomic-embed-text tool calling host process OpenAI-compatible

Gmail SMTP
smtp.gmail.com:465 · App Password · smtplib
LLM-drafted alert emails delivered via Gmail App Password. Uses Python smtplib — zero external service dependencies. Configured via GMAIL_USER, GMAIL_APP_PASSWORD, and ALERT_RECIPIENT in .env.
